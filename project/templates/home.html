Introduction

Contemporary bioinformatics challenges require agile event-driven pipelines to query, cleanse, and analyze data in support of scientific inquiry. This API handles the process of fetching data from various biological data sources and provides standardized output for analytics pipelines.

Canine was developed as a proof of concept for how event-driven microservice patterns powered by Python Flask APIs in conjunction with Kubernetes containerized clusters can revolutionize the study of biology.

Tutorial

End points are added to the primary URL of the API to instruct the system to perform a given task. In this case, add an endpoint to the base URL: http://localhost:8000/... to access its functionality.

API

The API endpoint will connect to a resource and return the specified output according to the argument(s) passed to it. The currently supported arguments are presented below:

  • version - sets what version of the API should be used.

    • .../api?version="0.0.0"

  • user - logs which user is accessing this service by recording their email.

    • .../api?user="skynet@world.domination"

  • resource - determines the target resource for fetching data.

    • .../api?resource="ncbi"

  • database - selects which database of the resource will be queried.

    • .../api?database="nucleotide"

  • term - search term for the query formatted according to the guidelines of the resource or database.

    • .../api?term='CRT[Gene Name] AND "Plasmodium falciparum"[Organism]'

  • format - the format in which the returned files will be delivered.

    • .../api?format="gb"

Version History

This API is new, experimental, and is designed mostly as a proof of concept for event-driven microservices for bioinformatics tasks using Python Flask APIs and Kubernetes clusters.

Release cryptograms follow the names of domesticated canine breeds.

Corgi

The beta stage of development (0.X.X) for Canine has been assigned the cryptogram Corgi.

0.0.0

Initial Canine release. It has a single endpoint with a few basic arguments for controlling what data is fetched from its single source. Future releases will support additional databases and will gradually expand the number of supported arguments.
